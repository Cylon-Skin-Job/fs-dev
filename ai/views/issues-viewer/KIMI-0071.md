This is a reminder to handle the macOS **Full Disk Access** requirement before packaging the application for deployment. The new direct SQLite connection for the Calendar requires this permission to read `~/Library/Calendars/Calendar Cache`.

### Required Actions:

1.  **Test Packaging:** When the app is packaged into a `.app` bundle, it will need to be manually added to **System Settings > Privacy & Security > Full Disk Access** by the user.

2.  **Implement Permission Check:** The `SchemaDetector` or the initial database connection logic should catch `EPERM` (Permission Denied) errors.

3.  **Build Guidance UI:** If a permission error is detected on startup, the app should show a user-friendly modal that:
    *   Explains *why* the access is needed (e.g., "for high-performance calendar access").
    *   Provides a button that opens the correct System Settings pane.
    *   Includes a screenshot or GIF showing the user exactly what to do.
