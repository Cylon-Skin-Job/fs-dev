# Chunk 2: Server CSS segmentation

**From:** THEME_SYSTEM_REFACTOR_SPEC.md §5.2  
**Goal:** Break the monolithic `renderSlugToCss` into focused segment modules plus a thin orchestrator. Strip `themes-service.js` to file I/O + CRUD only.  
**Depends on:** Chunk 1 (color-math.js exists)  
**Parallel with:** Chunk 3 (override system) — Chunk 2 is pure refactor, Chunk 3 adds new behavior. No file overlap.

---

## Context

`themes-service.js` currently has `renderSlugToCss()` — a 138-line function that generates the entire `:root` CSS block. It computes surfaces, text, borders, accents, workspace tints, and syntax highlighting all in one place.

This chunk splits that function into segment modules, each responsible for one visual area. The orchestrator combines them into the final CSS string. **No behavior change.** The output must be byte-identical for the same input.

---

## Files to create

### `open-robin-server/lib/theme/theme-css-generator.js`

**Job:** Orchestrator. Imports all segment modules, calls each `render(entry)`, combines fragments into the final `:root { }` block.

```js
const panelCss = require('./panel-css');
const contentCss = require('./content-css');
const textCss = require('./text-css');
const borderCss = require('./border-css');
const accentCss = require('./accent-css');
const workspaceCss = require('./workspace-css');
const syntaxCss = require('./syntax-css');

function render(entry) {
  const fragments = [
    panelCss.render(entry),
    contentCss.render(entry),
    textCss.render(entry),
    borderCss.render(entry),
    accentCss.render(entry),
    workspaceCss.render(entry),
    syntaxCss.render(entry),
  ];

  return `/* Generated by themes-service.js — do not edit; regenerated on theme switch. */
:root {
${fragments.join('\n')}
}`;
}

module.exports = { render };
```

**Target:** Under 80 lines.

### `open-robin-server/lib/theme/panel-css.js`

**Job:** Panel and chrome surfaces.

**Returns CSS fragment for:**
```css
  --bg-solid:              ...;
  --bg-primary:            ...;
  --bg-secondary:          ...;
  --panel-chrome-bg:       ...;
  --sidebar-surface-bg:    ...;
  --chat-surface-bg:       ...;
  --neutral-chrome-bg:     ...;
  --panel-bg:              ...;
  --robin-card-bg:    ...;
  --robin-card-hover: ...;
  --robin-input-bg:   ...;
  --robin-chat-bg:    ...;
  --robin-border:     ...;
```

**Implementation notes:**
- Receives `entry`, computes `luminance`, `panelContrast`, `bgTint`, `accent`
- Computes surface stack: `floorL`, `surfaceL`, `codeL`, `panelL`
- Uses `surface()` helper and `luminanceToHex()` — include these locally in this file
- `cardBg = surf` (same as surface)
- `--robin-border` uses `neutralBorder` — either recompute it locally or import from `border-css.js`
  - **Recommended:** recompute `neutralBorder` locally to keep the module self-contained. It's a one-liner.

### `open-robin-server/lib/theme/accent-css.js`

**Job:** Accent family, chrome accent, and per-CLI/tile/Robin aliases.

**Returns CSS fragment for:**
```css
  --theme-primary:     ...;
  --theme-primary-rgb: ...;
  --theme-border:      ...;
  --theme-border-glow: ...;
  --ws-primary:        ...;
  --ws-primary-rgb:    ...;
  --chrome-accent:     ...;
  --cli-accent:    ...;
  --tile-color:    ...;
  --robin-primary: ...;
```

**Implementation notes:**
- Computes `r, g, b` from `accent` via `hexToRgb`
- `--ws-primary` and `--ws-primary-rgb` are `var(--theme-primary)` aliases
- `--chrome-accent` is `color-mix(in srgb, ${accent} ${chromeTint}%, rgba(0, 0, 0, 0.4))`
- `--cli-accent`, `--tile-color`, `--robin-primary` are `var(--chrome-accent)` aliases
