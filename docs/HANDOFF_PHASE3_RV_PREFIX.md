# Handoff: Phase 3 — `.rv-` class prefix migration

**Roadmap:** `docs/CODE_STANDARDS_CLEANUP_ROADMAP.md`
**Baseline commit:** `7f74ce9` (Phase 2 complete, pushed to origin/main)
**Goal:** Every custom CSS class in source CSS files must be prefixed `.rv-`. This prevents collision with third-party libraries and makes origin-of-class obvious when reading TSX. Each rename is CSS-definition + all TSX usages in one atomic commit.

**Two CSS layers that both need updating:**
- **App layer:** `fusion-studio-client/src/**/*.css` + corresponding TSX
- **Project layer:** `ai/settings/*.css` → classes are referenced in TSX but served at runtime. When you rename in the project-layer CSS, also rename in the Fusion-Home copy at `~/projects/Fusion-Home/ai/settings/` (disk only, no git commit there).

**Exempt classes — do NOT rename these:**
- `.token-*` — generated by Prism.js syntax highlighter; renaming breaks highlighting
- `.hljs-*` — generated by highlight.js; same reason
- `.material-symbols-outlined` — Google Fonts library class
- `::-webkit-scrollbar`, `::-webkit-scrollbar-*` — pseudo-elements, not classes
- `body[data-*]` — attribute selectors, not classes

---

## Operating rules (non-negotiable)

1. Before touching any file, read it from disk. Never work from memory.
2. For each class being renamed: grep the TSX source for every occurrence of the bare string before editing. If you miss a TSX usage, the browser will silently lose that styling.
3. CSS rename + all TSX usages in the same commit. Never split them.
4. Build must pass after each chunk before committing.
5. After all chunks, write a one-paragraph summary of what changed and what (if anything) didn't match expectations.
